Learning syntacsis basis Java Script

  • Вариант №1
    console.log('Учим JS')
    ошибку не вызовит, но лучше закрыть точкой с запятой (;)
  • Вариант №2
    console.log('JS')
    console.log('Учим')
    ошибку не вызовит (так как начитается с новой строки), но лучше закрыть точкой с запятой (;)
  • Вариант №3
    console.log('Учим') console.log('JS')
    вызовит ошибку (так как в одном рядке), нужно закрыть точкой с запятой каждую команду (;)
  • Вариант №4
    console.log('JS');
    console.log('Учим');
    ошибку не вызовит (так как закрыто ;), все отлично.

    let user = 'Вася' let age = 36
    ошибку вызовит отсутствие точкой с запятой (;) между переменными в одной строке

    let myage = 36;
    ошибку вызовит отсутствие camelStyle в написание названия переменных

    • Learning Java Script data types and data structures

  • Вариант №1
    let userAge = 36;
    let userInfo = 'Фрилансер ${userAge}';
    ошибку вызовит не правильное написание обекта userInfo. Правильное:
    let userAge = 36;
    let userInfo = { Фрилансер: userAge};
  • Вариант №2
    let userHeight = 145 / 0;
    Не верно, userHeight вернет значение Infinity(выполняется деление, это выражение больше любого числа, он является верным в отличии от NaN), тип данных number
  • Вариант №3
    let userName;
    Тип данных undefined так как значение не заданно или является не определенным в отличии от null(пустота)
  • Вариант №4
    let userSize - "45" / "8";
    Тип данных number, хоть значение и задано как строчное, но значения числовые и оператор деления преобразовывает их в числовые
